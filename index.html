#DOM

DOM capture and bubbling; controls

capture
traverse down from the outermost html tag to the innermost html tag where an event is called.

bubbling
traverse up from the innermost html tag to the outermost html tag where an event is called.

event.stopPropagation()
a control to prevent bubbling from occurring beyond where the event is initially called.

capture: true
option within an .addEventListener to initialise the event call in the capture direction rather than the bubbling direction where the node presides.

once: true
option within an .addEventListener to remove the event listener from the node object.

#DOM

Node properties: .innerText, .textContent and .innerHTML

for a div element containing two child elements kbd and span:

.innerText
returns the visible text, "A\nCLAP"

.textContent
returns the full text, "\n      A\n      clap\n    "

.innerHTML
returns "\n      <kbd>A</kbd>\n     <span class=\"sound\">clap</span>\n    "

.innerText has performance issues, use textContent where possible.

if textContent cannot be used easily, consider changing the HTML structure to suit the call better

#DOM

Objects in event handling

btn.addEventListener('click', function (event) { return event; }
the argument passed into the function represents the event object; the associated properties of the event object can then be accessed.

btn.addEventListener('click', function (event) { return event.target; }
event.target looks at the event object's 'target' property; i.e. it returns the object listening for the event; i.e. the html element.

#DOM

Difference between DOM and HTML

HTML is a markup language; the .html extension are for files written in HTML; these files are read by the web browser and rendered as elements.

DOM is an interpretation of the html file as an object to be interacted with; nodes within the DOM can be manipulated which then reflect in HTML elements on the web page.

The DOM is an interface between javascript and html + css; it enables javascript to change the html and css during run time.

#script

Using async and defer in the script tag when loading javascript file

no option
the html loads; html loading pauses while js script is fetched; when the js file is fetched it's executed, then the html file unpauses and continues loading.

defer
the html loads; the js script is fetched at the same time; the js file executes only after the html fully loads.

async
the html loads; the js script is fetched at the same time; when the js file fetch finishes, the js file is executed immediately.

#DOM

Relationships in the Document Object Model: ancestor, parent, child, sibling.

  • div.class
  • .class div
  • .class > div

div.class
selects a div with the mentioned class.

.class div
selects a div whose ancestor element belongs to the mentioned class.

.class > div
looks for elements that have the mentioned class and selects div elements that are children to it
